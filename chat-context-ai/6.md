## Containerizing `linera-service`

The Linera project provides a `Dockerfile`  that can be used to build a Docker image containing the `linera` binaries, including `linera-service` .

The general steps to containerize `linera-service` are:

1.  **Build the Docker Image**: The `docker/Dockerfile`  defines how to build the Linera binaries into a Docker image. This Dockerfile supports building the `linera` binary, which includes the `linera service` command .
    You can build the image using a command similar to what's found in the `docker_image.yml` GitHub Actions workflow  or the `compose.sh` script :
    ```bash
    docker build --build-arg git_commit=$(git rev-parse --short HEAD) -f docker/Dockerfile . -t linera
    ```
    This command builds the Docker image named `linera` using the `docker/Dockerfile` . The `git_commit` build argument is used for versioning information inside the binaries .

2.  **Run the Docker Container**: Once the image is built, you can run a container from it. The `linera service` command requires a `--port` argument .

    An example of running `linera service` in a Docker container would be:
    ```bash
    docker run -p 8080:8080 linera linera service --port 8080
    ```
    This command runs the `linera` image, executes the `linera service --port 8080` command inside the container, and maps port `8080` of the container to port `8080` on the host.

## Exposed Ports

The `linera service` command exposes a GraphQL service . The port for this service is specified using the `--port` option .

*   **`--port <PORT>`**: This option specifies the port on which the GraphQL server will run . The default value for this port is `8080` when called from `run_node_service` .

Therefore, to access the `linera service` from outside the Docker container, you need to expose the port specified by the `--port` argument. If you use the default `8080`, then port `8080` needs to be exposed.

For example, in a `docker-compose.yml` file, you might see services like `proxy` and `shard` that expose ports. The `proxy` service, for instance, exposes port `19100` . While this is not directly for `linera service`, it demonstrates how ports are exposed for Linera components in a Docker Compose setup.

## Key Files for Linera Wallet Management

The Linera CLI tool uses specific files to manage your wallet and keys  .

*   **`wallet.json`**: This file contains the private state of your user chains . It's created by the `linera wallet init` command . The default location for this file is typically within a `linera` subdirectory in your system's configuration directory, or it can be specified via the `LINERA_WALLET` environment variable or the `--wallet` command-line option .
*   **`keystore.json`**: This file stores the keystore state, which includes your private keys . Similar to `wallet.json`, its location can be configured via the `LINERA_KEYSTORE` environment variable or the `--keystore` command-line option, with a default location next to `wallet.json` .

Additionally, the `LINERA_STORAGE` environment variable or `--storage` option specifies the storage configuration for the blockchain history . While not directly part of the wallet keys, it's crucial for the client to connect to the correct blockchain data .

## Exporting and Backing Up Wallet Keys

To export and back up your Linera wallet keys, you need to copy the `wallet.json` and `keystore.json` files .

1.  **Locate the files**:
    *   The `linera` CLI tool will output the paths to `LINERA_WALLET`, `LINERA_KEYSTORE`, and `LINERA_STORAGE` when a local test network is started .
    *   By default, these files are located in a `linera` subdirectory within your system's configuration directory . For example, in a local test network setup, they might be in `/tmp/local-linera-net/`  .
    *   If you are using Docker, scripts like `scripts/backup-validator-keys.sh` and `scripts/extract-wallet-from-volume.sh` are designed to locate and copy these files from Docker containers or volumes  .

2.  **Copy the files**: Once located, simply copy `wallet.json` and `keystore.json` to a secure location .

## Importing and Restoring a Linera Wallet

To restore a Linera wallet on another machine or after a system reset, you need the backed-up `wallet.json` and `keystore.json` files .

1.  **Place the files**: Copy your backed-up `wallet.json` and `keystore.json` files to the desired location on the new machine .
2.  **Specify file paths**: When running Linera commands, you can specify the paths to these files using:
    *   The `--wallet` and `--keystore` command-line options .
    *   The `LINERA_WALLET` and `LINERA_KEYSTORE` environment variables .
    *   For example:
        ```bash
        export LINERA_WALLET="/path/to/your/wallet.json" 
        export LINERA_KEYSTORE="/path/to/your/keystore.json" 
        linera wallet show
        ```
3.  **Configure storage (if necessary)**: Ensure that the `LINERA_STORAGE` environment variable or `--storage` option is correctly set to point to your blockchain's storage backend . This is crucial for the wallet to interact with the blockchain .

## Notes

The `linera-service/src/cli/main.rs` file defines how the `linera` CLI tool determines the paths for `wallet.json` and `keystore.json`, prioritizing command-line arguments, then environment variables, and finally default configuration directories . The `scripts/backup-validator-keys.sh` and `scripts/extract-wallet-from-volume.sh` scripts provide practical examples of how these files are handled in a Dockerized environment for backup and extraction purposes  . The `Makefile` also demonstrates setting `LINERA_WALLET`, `LINERA_KEYSTORE`, and `LINERA_STORAGE` environment variables for development workflows .
